## DESIGN SYSTEM REQUIREMENTS

### Style Direction
- **Modern App-Like Interface**: Not a landing page, but a functional application
- **Glassmorphism**: Subtle glass effects with backdrop blur
- **Clean & Professional**: Enterprise-grade design
- **Minimal but Rich**: Clean layouts with meaningful content
- **Smooth Animations**: Subtle transitions and micro-interactions
- **Depth & Hierarchy**: Clear visual hierarchy using shadows, spacing, and typography

### Typography
- **Font Family**: Inter (sans-serif)
- **Headings**: Bold, navy color (#0C1F3F)
- **Body Text**: Regular weight, gray color (#6F7D94)
- **Line Heights**: Comfortable (1.4-1.6)
- **Letter Spacing**: Slightly negative for large headings (-0.01em to -0.02em)

### Spacing System
- Consistent spacing scale: 4px base unit
- Generous whitespace for breathing room
- Card padding: 24-32px
- Section spacing: 48-64px

### Border Radius
- Cards: 24-32px (rounded-glass)
- Buttons: Full rounded (pill shape)
- Inputs: 16-24px
- Small elements: 8-12px

## PAGES & ROUTES

### 1. Homepage (/)
**Purpose**: Landing page for non-authenticated users
**Components**:
- Hero section with title, subtitle, CTA buttons
- Quick stats (3 cards: 25K+ placements, 180+ partners, 4 languages)
- Job highlights (grid of job cards, 6 items)
- Quick features (3 feature cards with icons)
- UI mockups section (desktop dashboard + mobile app preview)

**Requirements**:
- App-like, not marketing-heavy
- Focus on functionality preview
- Show platform capabilities visually

### 2. Login Page (/login)
**Purpose**: User authentication
**Components**:
- Split layout: Left side with logo and branding, Right side with form
- Email input field
- Password input field
- Login button (primary orange)
- Link to register page
- Error message display
- Loading state

**Requirements**:
- Clean, professional form design
- Clear error states
- Smooth transitions

### 3. Register Page (/register)
**Purpose**: New user registration
**Components**:
- Split layout similar to login
- Form fields: Name, Email, Phone, Password, Languages (comma-separated)
- Register button
- Link to login page
- Error message display
- Loading state

### 4. Jobs Listing Page (/jobs)
**Purpose**: Browse all available jobs
**Components**:
- Page header with title
- Job cards grid (responsive: 1 col mobile, 2 col tablet, 3 col desktop)
- Each job card shows: Title, Location, Salary, Required Language, Required Experience, Description preview
- "Apply Now" button on each card
- Filter/search functionality (optional but recommended)

**Requirements**:
- Easy to scan job listings
- Clear hierarchy
- Hover effects on cards

### 5. Job Detail Page (/jobs/[id])
**Purpose**: View full job details
**Components**:
- Job title (large heading)
- Job details: Location, Salary, Experience, Language requirements
- Full description
- Apply button (prominent, orange)
- Back to jobs link

**Requirements**:
- Clear, readable layout
- Prominent CTA button
- Easy to understand requirements

### 6. User Dashboard (/user/dashboard)
**Purpose**: Main workspace for candidates
**Components**:
- Welcome section with user name
- Metrics cards (4 cards): Applications count, Live jobs count, Unread notifications, Languages count
- Two-column layout:
  - Left: Available jobs list (scrollable)
  - Right: Application form (CV upload, Certificate upload, Job selector, Submit button)
- Application tracking section (list of submitted applications with status)
- Notifications panel (list of notifications with read/unread states)

**Requirements**:
- Dashboard-like interface
- Clear data visualization
- Easy file upload interface
- Status indicators (colors: green for active, orange for review, gray for pending)

### 7. Recruiter Dashboard (/recruiter/dashboard)
**Purpose**: Workspace for recruiters
**Components**:
- Welcome section
- Metrics cards: Active jobs, Current applicants, Countries hiring
- Two-column layout:
  - Left: Create new job form (Title, Description, Location, Salary, Experience, Language)
  - Right: Job list with applicant counts
- Applicants view (when job selected): List of applicants with their CVs, status management

**Requirements**:
- Professional recruiter tools
- Easy job creation
- Clear applicant management

### 8. Admin Dashboard (/admin/dashboard)
**Purpose**: System administration
**Components**:
- Welcome section
- Metrics cards: Jobs, Recruiters, Users, Logs
- Admin user management
- Jobs overview
- System logs display

**Requirements**:
- Clean admin interface
- Data-heavy but organized
- Clear system status

### 9. About Page (/about)
**Purpose**: Company/platform information
**Components**:
- Mission statement
- Vision statement
- Values list
- Team information (optional)

### 10. Contact Page (/contact)
**Purpose**: Contact form
**Components**:
- Contact form: Name, Email, Message
- Submit button
- Success message

## USER ROLES & PERMISSIONS

1. **User (Candidate)**: Can browse jobs, apply, upload CV/certificates, view applications
2. **Recruiter**: Can create jobs, view applicants, manage applications
3. **Admin**: Can manage users, jobs, view system logs
4. **Superadmin**: Full system access

**Visual Differentiation**:
- Role badges/indicators where appropriate
- Different dashboard layouts per role
- Role-specific color accents (optional)

## MULTILINGUAL SUPPORT

**Supported Languages**: English (EN), Turkish (TR), French (FR), Arabic (AR)

**Requirements**:
- RTL support for Arabic (right-to-left layout)
- Language switcher in header
- All text must be translatable
- Icons and images should be language-neutral
- Date/number formats should respect locale

## RESPONSIVE DESIGN

**Breakpoints**:
- Mobile: < 640px (1 column layouts)
- Tablet: 640px - 1024px (2 column layouts)
- Desktop: > 1024px (3+ column layouts, full features)

**Requirements**:
- Mobile-first approach
- Touch-friendly buttons (min 44x44px)
- Readable text sizes on mobile
- Collapsible navigation on mobile
- Optimized images for all screen sizes

## ANIMATIONS & INTERACTIONS

**Micro-interactions**:
- Button hover: Scale up (1.05x), shadow increase
- Card hover: Lift up (-4px), shadow increase
- Input focus: Border color change, ring effect
- Page transitions: Fade in (0.6s ease-out)
- Loading: Smooth spinner rotation

**Animation Principles**:
- Subtle, not distracting
- Performance-optimized (use transform/opacity)
- Respect user preferences (prefers-reduced-motion)

## UI COMPONENTS SPECIFICATIONS

### Buttons
**Primary Button**:
- Background: Brand Orange (#F47B20)
- Text: White
- Shape: Fully rounded (pill)
- Padding: 32px horizontal, 12px vertical
- Hover: Scale 1.05, shadow increase, slight lift
- Active: Scale back to 1.0

**Secondary Button**:
- Background: Transparent
- Border: Brand Blue (#1D6EEA) with 20% opacity
- Text: Brand Blue
- Shape: Fully rounded
- Hover: Background tint, border opacity increase

### Input Fields
- Background: White with 90% opacity
- Border: Slate gray with 80% opacity
- Border radius: 16-24px
- Padding: 16px horizontal, 12px vertical
- Focus: Blue border, ring effect, shadow
- Placeholder: Brand Gray color

### Cards
- Background: White with 80-90% opacity
- Border: White with 40-60% opacity
- Border radius: 24-32px
- Shadow: Soft shadow (see shadows above)
- Padding: 24-32px
- Hover: Shadow increase, slight lift

### Badges/Tags
- Background: Colored with 10-20% opacity
- Text: Matching color (darker)
- Shape: Fully rounded (pill)
- Padding: 8px horizontal, 4px vertical
- Font size: 12-14px
